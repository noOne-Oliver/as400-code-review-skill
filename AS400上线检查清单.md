# AS400 上线前检查清单
# 适用场景：项目上线前 4 天质量保障

---

## 🔴 P0 - 必须修复（上线阻断项）

### 变量初始化检查

- [ ] 所有数字变量（P/S 类型）都有 INZ 或明确赋初值
- [ ] 字符变量有 INZ(*BLANKS) 或明确初始化
- [ ] 指针变量有 INZ(*NULL)
- [ ] DataStructure 的每个子字段都单独初始化
- [ ] 数组声明有 INZ 或在使用前显式清空

### 文件操作检查

- [ ] 所有 CHAIN/READ/SETGT 操作后检查 %FOUND/%EOF
- [ ] 所有 COMMIT 操作都有对应的 ROLLBACK 测试
- [ ] 有锁操作的地方有超时处理（不会无限等待）

### 编译检查

- [ ] 编译无 Warning（或 Warning 都已确认无影响）
- [ ] 所有 DS 大小与外部定义匹配

---

## 🟡 P1 - 应该修复（影响业务）

### 变量命名检查

- [ ] 无 TMP1/TMP2/X/XX 等无意义临时变量名
- [ ] 无名实不符的变量（如变量名是 CUSTNAME 但存的是地址）
- [ ] 数字变量有单位标注（AMT_YUAN / AMT_CENT / QTY 等）
- [ ] 无魔法数字（999/00 等硬编码常量已提取为命名常量）
- [ ] FLAG 的含义已文档化（0/1 还是 A/B）

### 业务逻辑检查

- [ ] 除法操作前检查除数不为 0
- [ ] SETLL 操作明确知道是精确匹配(%EQUAL)还是近似匹配
- [ ] 字符串拼接使用 %TRIM 避免多余空格

### 日志与错误处理

- [ ] 关键操作有日志记录（入口/出口/异常）
- [ ] 错误有明确的处理路径（非静默失败）

---

## 🟢 P2 - 建议优化（可延后）

### 代码质量

- [ ] 单字符变量名有明确用途（K 除外，那是 RPG 的键命名）
- [ ] 关键逻辑有注释说明
- [ ] 无重复代码（DR (Definition) 或 Copy 代码复用）

### 性能检查

- [ ] 循环内无不必要的文件 I/O
- [ ] 批量操作考虑用 SQL 替代逐条 RPG 处理

第一轮（核心交易）：
1. 订单处理程序 (ORD*)

第二轮（辅助流程）：

## ⚡ 快速扫描命令（人工执行）

在 SEU 或 RDi 中搜索以下模式：

### 变量未初始化
```
未找到 "INZ" 的 "D xxx S nnP"
未找到 "INZ" 的 "D xxx S *"
```

### 潜在除零
```
找到 " / " 后检查除数是否有为零判断
```

### 未检查的 CHAIN
```
找到 "CHAIN" 后 5 行内应有 %FOUND 检查
```

### 魔法数字
```
找到 "999" "00" "9999" 等可疑常量
```
